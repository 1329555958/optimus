定义

  • A方: 上海富捷信息技术有限公司
  • B方: 福建国通星驿网络科技有限公司
  • 借款人: 标的申请人
  • A方账户: B方提供的供A方操作的共管账户
  • B方账户: B方的实际银行账户
  • 本系统: B方用于日常运营的后台管理系统，用户名密码由A方提供

出入款说明

  • 出款: 从A方账户转账到借款人账户
  • 入款: 借款人转账到A方账户
  • 渠道出款: 从B方账户转账到A方账户
  • 渠道入款: 从A方账户转账到B方账户
  • 出入款条目: A方出款包括产品的放款、基金申购、账户提现，入款包括产品的还款基金的赎回；B方目前只有一种出款类型，就是账户充值;

项目运行流程

  1. B方提供理财产品，在A方理财平台后台系统上线、审核
  2. B方在本系统中的定期产品管理对通过A方审核的理财产品进行上架
  3. B方会员在APP中充值、购买
  4. 产品满标后，B方需要发起一笔渠道出款操作，然后根据转账凭证号，在本系统中的 待出入款管理 发起渠道出款申请，A方会根据凭证号进行审核
  5. A方通过审核后，会发起一笔出款操作，然后根据转账凭证号，通过A方后台系统发起出款申请，B方在本系统中的 审核 进行审核
  6. 等待投资期限结束
  7. 借款人还款到A方账户,A方根据借款人的还款凭证号，发起入款申请，B方在本系统中的 审核 进行审核
  8. B方可发起一笔渠道入款操作，然后根据转账凭证号，在本系统中的 待出入款管理 发起渠道入款申请，A方会根据转账凭证号进行审核
  9. 一个理财产品的生命周期到此结束

产品管理

货币基金管理


说明:
配置货币基金相关信息；

  • 编号、名称: 由基金公司确定的，要确保准确无误
  • 起投金额：用户能购买的最小金额，
  • 七日年化收益率、每万份收益：这些随着日期而变化的，运营人员需要每天登陆基金公司官网查看并录入；如果为0，基金详情页面将不显示这些信息
  • 开放购买：只有开放购买了，投资用户才能看到并进行申购；否则投资用户看不到基金信息

货基交易记录


说明：
根据查询条件查询响应的申购赎回记录；默认查询最近一天的所有用户的交易记录；

  • 用户信息：输入完整的用户编号或者名称进行查询单个用户的交易记录
  • 交易类型：申购或者赎回
  • 交易时间：查询对应时间段内的交易记录

货基收益详情

说明:
根据不同条件， 查看货币基金的收益信息；

  • 用户信息：输入完整的用户编号或者名称，查看单个用户的收益信息
  • 收益状态：发放成功或者失败, 如果存在发放失败的记录，应该是系统出现了异常
  • 收益日：查看单个收益日的收益信息

定期产品管理


说明：
对固定期限的理财产品进行上下架管理，及详情、投资、还款等信息查看

  • 产品状态：未上架表示投资用户看不到此产品，已上架是等待产品处于招标中时，投资用户可以进行投资； 已开售即招标中，已满标即已售罄，还款中即产品进入投资期，等待到期后借款人还款，已完结即此产品的生命周期已结束，投资人也已经收到本息还款；
  • 点击对应产品中的 【详情】 按钮 查看对应产品的详细信息

待出入款管理


说明:
出入款就是需要A方、B方、产品方之间进行资金流动的操作；当产品放款、基金申购时需要A方进行出款操作，当产品到期还款、基金赎回时需要A方进行入款操作；这里只是记录资金信息流，不会对实际银行账户进行转账操作；
目前B方只有一种出款类型就是账户充值;
可以对多个记录进行合并后进行出入款操作；勾选响应的出入款记录后，【生成待执行记录】

账户操作


说明：
先执行实际银行账户之间转账，根据交易凭证号在这里发起出入款申请，等待对方审核; 大多数情况下是先在 待出入款管理生成待出入款记录，然后在这里选择一条记录进行执行，只有账户出现异常时才会不关联具体产品进行出入款操作;
这里的出款、入款是相对于B方银行账户来说的，即渠道出入款；出款即B方向A方账户转账，入款即从A方账户中转账到B方账户；

  • 备注：说明信息
  • 凭证号：银行转账时的交易凭证号，审核时的主要参考依据

账户变动一览

常用操作

如何对产品上下架？

基金在货币基金管理中管理是否开放购买，这里的开发购买操作是让用户能否看到此基金产品；

定期产品在定期产品管理中进行上下架，未上架的产品用户看不到，即无法投资；

出入款操作

本系统中的出入款操作只是信息流，用以记录、对账等; 不会进行实际银行账户的操作，运营人员需要对银行账户进行实际操作之后拿着交易凭证号在本系统中发起出入款申请;

运营人员需要每天查看本系统中的待出入款管理是否有需要出入款的条目; 产品满标后放款及基金申购要做出款操作，产品还款及基金赎回要做入款操作，可以多个条目合并生成一个待执行记录，将合并后的轧差额进行账户操作;

在账户操作中选择一条待执行记录进行出入款操作；如果没有待执行记录通常是不需要进行账户操作的，只有账户出现异常时需要；

进行账户操作后，就等待对方审核,对方审核通过后出入款操作就完成了；

基金的申购需要在1点之前进行操作，因为A方要审核后去基金公司申购

审核

主要根据交易凭证号去查看实际银行账户中是否存在此交易,金额是否正确，此金额对应的出入款条目是否正确; 如果正确就审批通过,如果不正确就驳回，联系对方运营人员，让其核对后重新发起操作；
